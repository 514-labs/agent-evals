Run a benchmark:

```bash title="Run a benchmark"
docker run \
  -e ANTHROPIC_API_KEY=sk-ant-... \
  -e OPENAI_API_KEY=sk-... \
  ghcr.io/514-labs/dec-bench:ecommerce-pipeline.dbt.claude-code.sonnet-4.v1.0.0
```

JSON hits stdout. Nothing else to install, configure, or deploy.

## How It Works

Every image encodes four variables:

```text title="Image tag format"
{scenario}.{harness}.{agent}.{model}.{version}
```

| Variable | What it is | Examples |
|----------|-----------|----------|
| **Scenario** | A data engineering problem with infrastructure, seed data, a prompt, and validation | `ecommerce-pipeline`, `streaming-ingest` |
| **Harness** | Tools pre-installed for the agent | `bare`, `dbt`, `mcp-postgres` |
| **Agent** | The AI coding agent | Claude Code, Codex, Aider, custom |
| **Model** | The LLM behind the agent | sonnet-4, opus-4, gpt-4.1 |

Each combination builds a distinct, immutable container. No runtime flags. The image tag *is* the configuration.
