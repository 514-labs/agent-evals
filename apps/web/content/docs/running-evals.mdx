DEC Bench `0.1` is a research preview. The fastest way to help is to install the CLI, run one Foo Bar scenario locally, and note anything confusing, broken, or missing in the workflow.

<FirstRunRequirements />

## Prerequisites

Before your first run, make sure you have:

- Git installed so you can clone the DEC Bench repository
- Docker installed and running locally
- an API key for the agent you plan to use

The default first-run flow uses `claude-code`, so export `ANTHROPIC_API_KEY` before you run. If you use Codex instead, export `OPENAI_API_KEY` or `CODEX_API_KEY`. If you use Cursor, export `CURSOR_API_KEY`.

## Quick Start

<FirstRunPath />

```bash title="Clone the repo and enter it"
git clone https://github.com/514-labs/agent-evals.git
cd agent-evals
```

```bash title="Install the CLI"
curl -fsSL https://decbench.ai/install.sh | sh
```

`https://decbench.ai/install.sh` is the stable installer URL served by the Vercel-hosted app.

```bash title="Export the default agent API key"
export ANTHROPIC_API_KEY=your-key-here
```

Run the first workflow from the repo root. These commands use the CLI defaults for `base-rt`, `claude-code`, `claude-sonnet-4-20250514`, and `v0.1.0`.

```bash title="Build, run, and inspect"
dec-bench build --scenario foo-bar-csv-ingest
dec-bench run --scenario foo-bar-csv-ingest
dec-bench results --latest --scenario foo-bar-csv-ingest
```

`dec-bench run` prints the exact `run_id` at the end of the run. `dec-bench results --latest` also prints `Run ID: ...` at the top of the detailed view.

<RunIdHint />

```bash title="Open audit with the real run ID"
dec-bench audit open \
  --scenario foo-bar-csv-ingest \
  --run-id <run-id-from-results>
```

The installer fetches matching release or preview binaries from GitHub releases in `514-labs/agent-evals` and verifies checksums when they are published with the release.

The CLI passes through the keys your agent needs. Export them before you run:

| Provider | Variable | Agents |
|----------|----------|--------|
| Anthropic | `ANTHROPIC_API_KEY` | Claude Code |
| OpenAI | `OPENAI_API_KEY` | Codex |

| Cursor | `CURSOR_API_KEY` | Cursor |

## CLI Workflow

Build a specific scenario image:

```bash title="Build a scenario"
dec-bench build \
  --scenario foo-bar-csv-ingest \
  --harness base-rt \
  --agent claude-code \
  --model claude-sonnet-4-20250514
```

Run that scenario:

```bash title="Run an eval"
dec-bench run \
  --scenario foo-bar-csv-ingest \
  --harness base-rt \
  --agent claude-code \
  --model claude-sonnet-4-20250514
```

List available scenarios:

```bash title="List foo-bar scenarios"
dec-bench list
```

Inspect completed runs:

```bash title="Inspect results"
dec-bench results --scenario foo-bar-csv-ingest
dec-bench results --latest --scenario foo-bar-csv-ingest
dec-bench results --run-id <run-id>
```

The detailed view points directly to stdout, stderr, trace, assertion log, session JSONL, and run metadata sidecars when they exist.

Open the audit UI for one run:

```bash title="Open localhost audit UI"
dec-bench audit open \
  --scenario foo-bar-csv-ingest \
  --run-id <run-id>
```

That command exports the run into audit bundles, starts or reuses the local web app, and opens the exact audit page.

## Matrix Runs

Use the CLI for matrix runs too:

```bash title="Run the full matrix"
dec-bench run --matrix --parallel auto
```

## Version Policy

`0.1` uses one version convention:

- crate version: `0.1.0`
- release tag: `v0.1.0`
- default image suffix in the CLI: `v0.1.0`
- preview install: `DEC_BENCH_INSTALL_VERSION=<preview-tag> curl -fsSL https://decbench.ai/install.sh | sh`

## Help Improve The Preview

- Run one scenario from start to finish and keep track of any friction in install, build, run, results, or audit.
- Compare agents, personas, or harnesses if you want to generate more useful benchmark data.
- If you want to collaborate on the benchmark itself, continue with [Add an Eval](/docs/add-eval/getting-started).

## Advanced Docker Usage

You can still run images directly if you need to:

```bash title="Run an image directly"
docker run \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  ghcr.io/514-labs/dec-bench:foo-bar-csv-ingest.base-rt.claude-code.claude-sonnet-4-20250514.v0.1.0
```

The image tag still encodes `{scenario}.{harness}.{agent}.{model}.{version}`, but the CLI is now the recommended path.
