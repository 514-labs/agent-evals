# Exported Reports

Exported reports evaluate whether agents can deliver scheduled or on-demand report artifacts that are accurate, secure, and operationally reliable.

## Feature Intent

Users should receive report exports that are:

- Correct and reproducible
- Delivered on time
- Access-controlled and auditable
- Stable across reruns and format variants

## Capability Boundaries

Qualifies:

- Generates exports from governed reporting logic
- Supports scheduling and on-demand generation
- Includes delivery confirmation and retry behavior
- Produces traceable metadata (run timestamp, version, owner)

Does not qualify:

- Manual one-off file dumps with no repeatable process
- Exports without permissions or audit records
- Inconsistent CSV/XLSX/PDF outputs from the same report definition

## Competency Dimensions

- Reproducibility and version control of report logic
- Delivery reliability and failure recovery
- Security and access control
- Auditability and compliance posture
- Consumer usability (format quality and metadata clarity)

## Level Rubric (L1-L5)

| Level | Expected Capability |
| --- | --- |
| L1 | Produces ad hoc exports with minimal controls and no repeatability guarantees. |
| L2 | Supports basic recurring exports but with weak observability and manual failure handling. |
| L3 | Delivers scheduled and on-demand exports with stable formatting, retries, and audit trails. |
| L4 | Operates reliable export pipelines with strict access controls, versioned logic, and SLA visibility. |
| L5 | Demonstrates production-grade delivery with policy enforcement, compliance-ready traceability, and strong consumer trust. |

## Required Evidence

- Report definition source and version history
- Schedule configuration and delivery SLA/SLO records
- Retry policy and incident logs for failed deliveries
- Permission model for who can generate and receive exports
- Audit logs containing requestor, dataset scope, and delivery result

## Failure Modes and Anti-Patterns

- Export output differs across reruns with same inputs and window
- Silent delivery failures without alerts
- Privileged data included due to weak row/column level controls
- Unbounded report queries causing frequent timeout failures

## Suggested Test Scenarios

- Re-run the same report window and compare output checksums
- Force downstream delivery failure and verify retries and alerts
- Validate role-based access by attempting cross-tenant export requests
- Compare format parity across CSV/XLSX/PDF for core metrics
