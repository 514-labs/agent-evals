Features define the product capabilities that agents are evaluated on. Each feature page uses the same competency template so scores are comparable across evaluations.

## How Features Are Evaluated

Every feature is assessed with the same structure:

- Intent and expected user outcomes
- Capability boundaries (what qualifies and what does not)
- Competency dimensions
- Level rubric (L1 to L5)
- Required evidence and artifacts
- Failure modes and anti-patterns
- Suggested test scenarios

## v0.1 Status

Feature-specific evaluations are not yet included in the v0.1 release. The current scenarios focus on core data engineering competencies (ingestion, schema design, query optimization, etc.) rather than product feature delivery.

## Competency Backbone

The framework combines:

- Data engineering lifecycle concerns from *Fundamentals of Data Engineering* (ingestion, transformation, serving, orchestration, governance, and observability)
- System qualities from *Designing Data-Intensive Applications* (reliability, scalability, maintainability, and correctness tradeoffs)

This keeps feature scoring focused on system outcomes instead of specific tools.
