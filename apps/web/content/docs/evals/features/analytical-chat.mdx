---
title: "Analytical Chat"
description: "Evaluate conversational analytics for correctness, safety, and trustworthy decision support."
---

Analytical chat evaluates whether an agent can answer business questions through conversational interfaces while preserving metric correctness, access controls, and explainability.

## Feature Intent

Users should be able to ask analytical questions in natural language and receive answers that are:

- Grounded in governed semantic definitions
- Correct or explicitly uncertain
- Permission-aware
- Explainable and auditable

## Capability Boundaries

Qualifies:

- Translates user intent into governed analytical queries
- Surfaces assumptions, filters, and metric definitions in responses
- Applies role and data access policies before returning results
- Abstains or asks follow-up questions on ambiguous or infeasible requests

Does not qualify:

- Free-form answers disconnected from trusted data sources
- SQL generation without validation or safety checks
- Responses that bypass permission constraints

## Competency Dimensions

- Semantic grounding to canonical metrics and dimensions
- Query correctness and abstention behavior
- Safety and permission enforcement
- Explainability of generated analysis and assumptions
- Multi-turn context handling and error recovery

## Level Rubric (L1-L5)

| Level | Expected Capability |
| --- | --- |
| L1 | Provides generic analytical responses with weak linkage to governed data definitions. |
| L2 | Produces basic query-backed answers but with limited ambiguity handling and weak controls. |
| L3 | Delivers mostly correct, explainable, and permission-aware responses for standard analytical tasks. |
| L4 | Handles complex multi-turn analysis with robust safeguards, abstention logic, and strong traceability. |
| L5 | Operates as a trusted analytical interface with high correctness, transparent reasoning, and policy-by-default behavior. |

## Required Evidence

- Mapping of prompts to semantic entities/metrics used
- Validation logs for generated query correctness and execution outcomes
- Access-control enforcement tests and denial-path coverage
- Response traces showing assumptions, filters, and confidence signals
- Human review workflow for high-risk or low-confidence outputs

## Failure Modes and Anti-Patterns

- Hallucinated metrics not defined in the semantic layer
- Confident but incorrect SQL for underspecified questions
- Leakage of restricted dimensions or row-level data
- Missing explanation for how answers were computed

## Suggested Test Scenarios

- Ask equivalent questions in different phrasings and compare consistency
- Submit ambiguous prompts and verify abstain-or-clarify behavior
- Test restricted data questions with different user roles
- Run benchmark prompts with known answers and measure correctness rate
