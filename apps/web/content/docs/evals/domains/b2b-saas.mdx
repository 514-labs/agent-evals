# B2B SaaS

B2B SaaS workloads center on account-level product analytics, subscription lifecycle tracking, and feature adoption monitoring across tenants.

## Core Data

- Product usage events (page views, feature invocations, workflow completions).
- Subscription lifecycle records (trial, activation, upgrade, downgrade, churn).
- Account and workspace dimensions, often with user-level rollups.
- Entitlement and plan metadata used to segment behavior by contract tier.

## Characteristic Challenges

- High-cardinality user and account dimensions increase storage and query pressure.
- Event schemas evolve over time, requiring strict versioning and compatibility rules.
- Tenant isolation and account hierarchy logic can complicate aggregation correctness.
- Lifecycle state transitions need consistent ordering for retention and revenue analytics.

## Evaluation Focus

- Build reliable event models that preserve user, account, and plan attribution.
- Design incremental pipelines for adoption and lifecycle metrics.
- Handle schema evolution without breaking historical analyses.
- Produce trustworthy account-level KPIs (activation, expansion, churn signals).

## Typical Outputs

- Daily active users by account and plan tier.
- Feature adoption funnels by cohort and workspace size.
- Subscription transition timelines and churn risk segments.
- Expansion or contraction dashboards for revenue operations.
