Domains define the internal analytics and data warehousing contexts used to evaluate agent performance on realistic data engineering work.

## Internal Analytics Coverage

These domain tracks focus on common warehouse patterns: event-heavy pipelines, transactional analytics, billing-grade metering, and mixed structured or semi-structured models.

Use these pages to understand domain-specific data shapes, operational constraints, and evaluation targets before running or authoring tasks.

## Domain Map

- [B2B SaaS](/docs/evals/domains/b2b-saas): Product usage events, subscription lifecycle, and feature adoption with high-cardinality user and account dimensions.
- [B2C SaaS](/docs/evals/domains/b2c-saas): User activity streams, content interactions, and session data at time-series scale.
- [UGC (User-Generated Content)](/docs/evals/domains/ugc): Posts, comments, reactions, and moderation signals with variable JSON-heavy schemas.
- [E-commerce](/docs/evals/domains/e-commerce): Orders, inventory, catalog, and customer behavior with correctness-sensitive joins.
- [Advertising](/docs/evals/domains/advertising): Impressions, clicks, conversions, and bid data under extreme write throughput.
- [Consumption-Based Infra](/docs/evals/domains/consumption-based-infra): API, compute, and storage metering pipelines where billing accuracy is critical.

## How To Use These Pages

Start with the domain that best matches your workload profile, then map its constraints to competency and feature evaluations.

For cross-domain scenarios, combine domain assumptions with feature-focused pages under [Features](/docs/evals/features).
