# B2C SaaS

B2C SaaS workloads emphasize user activity streams, content interactions, and session-level behavior at very large event volume.

## Core Data

- User activity streams across web and mobile clients.
- Content interaction events (views, likes, shares, saves).
- Session and device telemetry with timestamps and source context.
- Cohort and profile dimensions used for growth and retention analysis.

## Characteristic Challenges

- Massive event throughput drives partitioning, compaction, and retention strategy requirements.
- Time-series heavy datasets demand fast windowed queries over large ranges.
- Retention and reactivation analysis require accurate user identity stitching.
- Late or duplicated events can skew cohort calculations if not normalized.

## Evaluation Focus

- Model time-series event tables for efficient retention and engagement analysis.
- Implement robust deduplication and late-arrival handling.
- Build sessionization and cohort transformations that scale.
- Optimize recurring retention queries for predictable performance.

## Typical Outputs

- D1, D7, and D30 retention by acquisition cohort.
- Session depth and frequency trends by platform and geography.
- Content engagement distribution by user segment.
- Reactivation and churn indicators over rolling windows.
